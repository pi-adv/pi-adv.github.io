Lesson 1 - The Basics

If you have done any research on learning a program language, you know that the first few things that they introduce is variables. I'll use the same format but I'll try an make it a little less redundent. Fortunately bash does not have a complicated system for variables.

Variables

You can define all sorts of variables in bash, anywhere from strings to numbers, you can store it in a variable. Variables are containers that you create and add values to it to use later. If you have used python, it will be the most similar to that language. What's great is that you don't need to define the type of variables if you want to create one. Though bash does have some stipulations when it comes to creating a variable and calling it.

  • Creating a Variable

    • Making a variable has a specific format.

    • First is the set the name of the variable on a new line.
      • test_variable

      *Be aware that you have to follow the rules as you would for python. No special character at the start, you can't have a command as a variable, and you cannot end the variable with a space, or even have a space in the variable.

    • Second, place an equal sign directly after, with no spaces in between.
      • test_variable=
    • Third is to put the value of the variable. Putting a number, or putting text in between brackets, or even a boolean value. But you can't have any space between the value and the equal sign.
      • test_variable=31 test_variable="July" test_variable=TRUE
  • Calling a Variable

    • Calling a variable is a little different than usual.

    • First, create a variable.

      • *If you don't have the variable you are calling it will error out.

      test_variable=31
    • Second, call the variable.

      • To call a variable you will need a $ symbol before the variable name.

      $test_variable
Setting up a File

Usually you don't need to do too much when setting up a file but there are some things that should be mentioned. First thing is the shabang. The shabang is a line in the file telling it where everything that bash needs is. This will typically be in your /bin/bash directory. Setting up a shabang is as simple as putting #!/bin/bash on the first line of any bash program you write. Another thing that you should pay attention to is the file extension. This is more for linux, but shell scripts can be made without the .sh extension at the end of the file. You can simply create a file without an extention and edit it as if it was a shell script. On Windows it will detect that it is a shell script or repopulate the .sh extention to the file. Then again Windows would be better off using .ps1 which is powershell.

Functions

I bring functions up because I had to search up how to do functions. Functions weren't as clear than I was thinking but they involve a good seamless process. To build a function you will need the following:

  • A Name for the Function.
  • Brackets to setup arguments.
  • Curly brackets to place the contents of the function.

So in practice here is an example:

startup (#args here) { echo "Hello World" }

Calling functions are as simple as calling the name of the function. For example:

if [] then startup fi

Lesson 2 - Commands

There are multiple commands that you can do in bash, some are only for bash and others are used in the linux terminal which you can call in a program. You are typically going to be using commands such as echo, read, clear. And statements like if, while, and for.

So let's go over a couple of statements and commands that are most common:

  • echo

    • The echo command is always used when you want to print or show something on the screen. All you need to do is call the echo command and then put what you want to echo to the screen.

    echo "Hello World!"
  • read

    • The read command is used when you want to read user input. You need to call the command and if you want to store that value, declare or call the variable after the command.

    read input_variable | or | read $input_variable
  • clear

    • The clear command is a personal preference for me, being that the clear command clears all lines on the terminal. It's used when you need a clean board to work on.

    clear
  • if statement

    • The if statement works just as you would with everyother language but with a few steps. In the brackets [] of the if statement you will put the statement for the if portion. The statement has to have a space on either side. Also an if/elif statement has to have a then statement after the arguments in the brackets (not applicable for the else statement). To end the if statement you need to put a fi statement to complete it. Ex:

    if [ 1=1 ] then echo "Hello!" elif [ 1=2 ] then echo "World!" else echo "!" fi
  • while statement

    • The while statement works by taking a set of parameters and continuing the intructions until the conditions are met. You use the do statement and the done statement when you are finished the while statement. Same with the if statements, you have to put the conditionals in brackets [] with spaces on either side. Ex:

    while [ 1=1 ] do echo "HELLO" done

Lesson 3 - Setting up a file to run

There are certain things that you will need to do to run your bash script in the terminal, first off is to load the terminal and change the directory to the directory that your script is located. Once done, run the change mod command and the +x condition, and the name of the file you are changing to execute. It should like like this: sudo chmod +x your_file.sh. If you run the list directory command, you should see that the file has changed colors. Now all you need to do is to run the file by doing: ./your_file.sh.

Files can be a little different, you can create a file with the .sh extension or not and using the command above it will still run. Bash scripts aren't typically the largest so don't be worried if your 200 line program is only a couple 100 kilobytes.
